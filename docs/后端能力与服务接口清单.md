# 后端能力与服务接口清单

本文档描述当前项目的后端能力边界。这里的“后端”不是独立 HTTP 服务，而是 Streamlit 页面直接调用的 Python 服务、调度器、数据源封装和 SQLite 持久化层。

## 1. 当前后端形态

当前项目的调用形态是：

- 前端页面：Streamlit 组件
- 中间层：业务服务对象 / 函数
- 数据层：SQLite、JSON 结果文件、运行时状态
- 调度层：后台线程/进程 + 持久化配置

### 7.5 自动执行规则

当前规则：

- A 股按一手 `100` 股执行
- 自动执行开启后，只要信号与账户条件都满足，系统会自动成交
- 规则不满足时会跳过，并写 `execution_note`

典型跳过原因：

- `缺少有效最新价`
- `建议仓位不足买入一手`
- `当前无可卖持仓`
- `当前无可卖数量`

## 12. 当前服务边界与重构约束

当前真实调用链大多是：

`Streamlit 页面 -> Python 服务对象/函数 -> SQLite / 数据源 / 调度器`

后续重构时，建议优先保持下面这些边界不变：

1. “我的关注”和量化候选池仍然是两层不同对象
2. 量化模拟与历史回放继续共用同一套量化候选池
3. 发现股票 / 研究情报只负责产出结果并手工加入“我的关注”
4. 持仓分析、实时监控、AI盯盘继续作为独立工作线
