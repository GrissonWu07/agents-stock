# 后端能力与服务接口清单

本文档只记录当前代码已经存在的后端边界，不再沿用旧的 `watchlist.db / candidate_pool / portfolio_stocks.db` 架构描述。

## 1. 网关入口

- 运行入口：`app/gateway.py`
- API 与路由：`app/gateway_api.py`
- 上下文装配：`app/gateway/context.py`

当前单服务提供：

- SPA 静态页面
- `/api/v1/*` 快照接口
- `/api/v1/*/actions/*` 动作接口

## 2. 主要数据库

### `data/quant_sim.db`

职责：

- 统一股票池主表 `stock_universe`
- 实时量化账户和执行状态
- 策略配置

### `data/quant_sim_replay.db`

职责：

- 历史回放任务和结果

### `data/selector_results/*.json`

职责：

- 发现股票和研究情报的结果缓存

## 3. 股票池相关服务

### `WatchlistService`

文件：`app/watchlist_service.py`

当前真实职责：

- 操作 `stock_universe`
- 管理 `watched` 视角
- 手工添加股票时写 `basic_info_missing`

### `CandidatePoolService`

文件：`app/quant_sim/candidate_pool_service.py`

当前真实职责：

- 对 `stock_universe.quant_enabled` 的兼容封装
- 不是独立物理候选池数据库

### `PortfolioDB / PortfolioManager`

文件：

- `app/portfolio_db.py`
- `app/portfolio_manager.py`

当前真实职责：

- 在 `stock_universe` 上管理登记持仓字段和 `registered_position_enabled`

## 8. 历史回放

### 页面快照

- `GET /api/v1/quant/his-replay`
- `GET /api/v1/quant/his-replay/progress`
- `GET /api/v1/quant/his-replay/capital-pool`

### 主要动作

- `POST /api/v1/quant/his-replay/actions/start`
- `POST /api/v1/quant/his-replay/actions/cancel`
- `POST /api/v1/quant/his-replay/actions/delete`

当前没有：

- `actions/continue`

### 核心服务

- `app/quant_sim/replay_service.py`
- `app/quant_sim/replay_runner.py`

### 当前回放数据边界

- 股票范围来自 `stock_universe.quant_enabled`
- 结果写入 `quant_sim_replay.db`
- checkpoint 阶段只读已准备历史数据

## 9. 刷新与缓存

### 当前可确认已落地的行为

1. `stock_refresh_scheduler` 对 fresh runtime entry 直接命中本地。
2. `stock_refresh_scheduler` 对失败 entry 做冷却跳过。
3. `LocalMarketDataStore.fetch_latest()` 用于实时 quote TTL。
4. `LocalMarketDataStore.fetch_range()` 用于历史区间 K 线和指标。
5. 历史回放准备阶段走 local-first，并对空区间和空股票写负缓存。

### 当前仍保留的兼容点

1. `CandidatePoolService` 这个命名仍然存在。
2. `refresh-indicators` 这个动作名仍然存在。
3. 发现/研究结果仍先落 JSON，再由用户推进到股票池。

## 10. 当前不该再写进文档的旧口径

- `watchlist.db` 是主关注池数据库
- `candidate_pool` 是独立物理候选池主表
- `portfolio_stocks.db` 是当前持仓主库
- live-sim 默认 15 分钟
- his-replay 支持 continue 到 live-sim
