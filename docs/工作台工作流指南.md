# 工作台工作流指南

这份文档只描述当前代码已经实现的主流程。

当前主线是：

`发现股票 / 研究情报 / 手工录入 -> 工作台股票池(stock_universe 视图) -> 按标签分流 -> 持仓诊断 / 实时量化 / 历史回放`

## 1. 工作台现在是什么

工作台已经不是旧的“我的关注 + 量化候选池”双池入口。

当前工作台表格实际展示的是统一股票池视图，底层来自 `xuanwu_stock.db.stock_universe`，默认包含至少一类有效成员：

- `watched=1`
- `quant_enabled=1`
- `registered_position_enabled=1`

所以一只股票即使不是纯关注股，只要已经启用实时量化或登记持仓，也会出现在工作台股票池管理视图里。

## 2. 三种成员标签

一只股票在系统里可以同时带多个标签：

- `watched`：持续关注
- `quant_enabled`：纳入实时量化和历史回放默认扫描范围
- `registered_position_enabled`：纳入持仓诊断

工作台就是这三个标签的主要管理入口。

## 3. 发现股票和研究情报的角色

`发现股票` 和 `研究情报` 不再直接定义 live-sim / his-replay 的输入范围。

它们的职责是：

1. 生成一批候选结果。
2. 将用户选中的股票写入统一股票池。

这些页面的结果本身保存在：

- `data/selector_results/main_force.json`
- `data/selector_results/low_price_bull.json`
- `data/selector_results/small_cap.json`
- `data/selector_results/profit_growth.json`
- `data/selector_results/value_stock.json`
- `data/selector_results/research.json`

它们是结果缓存，不是交易账户状态，也不是历史回放结果库。

## 4. 工作台上的关键动作

当前工作台最重要的动作有三组：

1. `添加股票`
   - 手工把股票写入 `stock_universe`
   - 若基础信息未知，会写 `basic_info_missing=true`

2. `加入量化`
   - 批量把股票标记为 `quant_enabled=1`
   - `CandidatePoolService` 现在只是 `stock_universe.quant_enabled` 的兼容服务层

3. `登记持仓`
   - 批量把股票标记为 `registered_position_enabled=1`
   - 同时写入登记持仓字段

旧的 `刷新关注池报价` 动作已经停用，当前后端会直接返回错误，要求改用统一数据刷新链路。

## 5. 实时量化怎么接股票池

`/live-sim` 读取的是统一股票池里：

- `quant_enabled=1`
- 默认 `status=active`

UI 文案已经改成：

- `实时量化股票`

它不再应该被理解成独立 `candidate_pool` 数据库或独立物理表。

实时量化自己的状态仍然独立保存在 `xuanwu_stock.db`：

- `strategy_signals`
- `sim_positions`
- `sim_position_lots`
- `sim_trades`
- `sim_account`
- `sim_account_snapshots`
- `sim_capital_slots`
- `sim_lot_slot_allocations`

## 6. 历史回放怎么接股票池

`/his-replay` 在任务启动时冻结统一股票池里 `quant_enabled=1` 的股票作为“回放股票范围”。

当前行为：

1. 历史回放不再接续 live-sim 账户。
2. 只有 `start / cancel / delete` 三类动作。
3. 回放结果只写入 `xuanwu_stock_replay.db`。

核心结果表：

- `sim_runs`
- `sim_run_checkpoints`
- `sim_run_events`
- `sim_run_signals`
- `sim_run_signal_details`
- `sim_run_trades`
- `sim_run_snapshots`
- `sim_run_positions`

## 7. 持仓诊断怎么接股票池

`/portfolio` 读取的是统一股票池上的登记持仓字段，不再依赖独立 `portfolio_stocks.db` 主库。

它关注的是：

- `registered_position_enabled=1`
- 持仓数量、成本、备注
- 最近行情技术快照
- 最近 AI 分析结论

个股详情页路径是：

- `/portfolio/position/:symbol`

## 8. 当前调度和刷新口径

当前已落地的刷新口径：

1. 实时量化调度默认 `10` 分钟。
2. 行情 quote TTL 为 `2` 分钟。
3. local-first 命中时不重复远程刷新。
4. 远程失败会进入冷却。
5. 历史回放准备阶段按区间补历史数据，checkpoint 阶段只读准备结果。

这意味着工作台、实时量化、历史回放现在至少已经在口径上不再是三套随意刷新的分散流程。

## 9. 推荐使用顺序

1. 先在 `发现股票` 或 `研究情报` 选出目标。
2. 将股票写入工作台股票池。
3. 在工作台判断它属于：
   - 观察
   - 实时量化
   - 登记持仓
4. 再进入 `/live-sim` 或 `/his-replay` 做验证。

如果只是看研究结论，不必立即启用 `quant_enabled`。
