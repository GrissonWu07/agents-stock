# 股票数据流说明

Date: 2026-05-06  
Status: Current code snapshot

Reference spec: `docs/superpowers/specs/2026-05-05-stock-universe-refresh-architecture-design.md`

## 1. 目的

本文档只做一件事：把“当前代码已经实现的数据边界”说清楚。

如果你看到旧文档里还在说：

- `watchlist.db`
- `candidate_pool` 独立主表
- `portfolio_stocks.db`
- 历史回放接续 live-sim

都以本文为准。

## 2. 当前股票主表

当前股票主对象已经统一到：

- `data/xuanwu_stock.db`
- 主表：`stock_universe`

关键字段：

- `stock_code`
- `stock_name`
- `watched`
- `quant_enabled`
- `registered_position_enabled`
- `basic_info_missing`
- `latest_price`
- `latest_signal`
- `metadata_json`

补充说明：

- `WatchlistService` 现在实际操作的是 `stock_universe`
- `CandidatePoolService` 现在实际操作的是 `stock_universe.quant_enabled`
- 登记持仓也在同一张表上做标签和字段扩展

## 3. 三个业务视角

统一股票池上的三个核心视角：

| 视角 | 筛选条件 | 主要页面 |
|---|---|---|
| 关注 / 工作台视图 | `watched OR quant_enabled OR registered_position_enabled` | `/main` |
| 实时量化股票 | `quant_enabled=1` | `/live-sim` |
| 登记持仓 | `registered_position_enabled=1` | `/portfolio` |

历史回放也是从 `quant_enabled=1` 冻结股票范围，但结果单独写 replay 库。

## 4. 发现和研究结果

发现股票、研究情报不是股票主表。

它们只是结果缓存：

- `data/selector_results/*.json`

主要用途：

1. 页面展示最新策略结果。
2. 允许用户把结果写入统一股票池。
3. 通过 `stock_runtime_snapshot.json` 叠加最新价和名称。

## 5. 实时量化状态

实时量化的账户和执行状态仍然独立保存在 live DB：

- `strategy_signals`
- `sim_positions`
- `sim_position_lots`
- `sim_trades`
- `sim_account`
- `sim_account_snapshots`
- `sim_capital_slots`
- `sim_lot_slot_allocations`
- `sim_scheduler_config`

这部分数据不定义股票池范围，只定义实时量化账户状态。

## 6. 历史回放状态

历史回放结果独立保存在：

- `data/xuanwu_stock_replay.db`

主要表：

- `sim_runs`
- `sim_run_checkpoints`
- `sim_run_events`
- `sim_run_signals`
- `sim_run_signal_details`
- `sim_run_trades`
- `sim_run_snapshots`
- `sim_run_positions`

这部分数据不回写到 `sim_positions / sim_trades / sim_account`。

## 7. 当前刷新口径

### 实时量化

- 默认调度周期：`10` 分钟
- 只在交易时段运行
- quote TTL：`2` 分钟
- local-first
- 失败冷却生效

### 历史回放

- 启动任务时冻结 `quant_enabled=1` 范围
- 准备阶段 local-first 补历史 K 线和指标
- checkpoint 阶段只读准备结果
- 默认 30m intraday segment 为 `90` 天

### 基础信息缺失

手工加股票时如果没有名称和基础信息，会在 `stock_universe.basic_info_missing=1` 上留下标记，后续刷新任务可以据此补数。

## 8. 当前已经废弃的口径

这些口径在当前代码里已经不成立：

- “工作台只展示关注股”
- “量化池和持仓池是两套独立股票主表”
- “历史回放和实时模拟共用结果表”
- “历史回放 continue 到 live-sim”
- “实时量化默认 15 分钟”

## 9. 一句话判断法

当你不确定某个数据属于哪里时，用这条规则判断：

- 这是股票对象本身：看 `stock_universe`
- 这是 live-sim 账户状态：看 `xuanwu_stock.db` 的 `sim_*` 和 `strategy_signals`
- 这是 his-replay 任务结果：看 `xuanwu_stock_replay.db` 的 `sim_run_*`
- 这是发现/研究结果缓存：看 `selector_results/*.json`
