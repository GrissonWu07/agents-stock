# 移动端适配（<=1200px）设计规格（方案 B）

> 日期：2026-04-21  
> 范围：`ui/` 前端界面（仅响应式与交互呈现层）  
> 约束：不影响 PC 端（`>1200px`）视觉与交互

---

## 1. 背景与目标

当前项目已具备部分响应式能力，但在 `<=1200px`（平板+手机）存在表格裁切、工具栏溢出、操作按钮拥挤等问题，导致核心业务页面不可高效使用。

本次采用**方案 B（混合方案）**完成移动端改造：

1. 表格保留核心列。
2. 次要字段进入“行展开详情”。
3. 行操作收拢到“更多（⋯）菜单”。
4. 表格容器允许内部横向滚动（兜底）。
5. 工作台 `Next step` 在 `<=1200px` 下沉到底部。

---

## 2. 非目标（本次不做）

1. 不重做整套移动端独立页面（不做双套 DOM 架构）。
2. 不改后端接口与数据结构。
3. 不改 `>1200px` 的布局与交互行为。
4. 不新增复杂手势交互（滑动删除、拖拽排序等）。

---

## 3. 强约束（已确认）

1. 断点范围：适配 `<=1200px`（B）。
2. 表格策略：核心列 + 展开详情（B）。
3. 行操作：收敛到更多菜单（B）。
4. `Next step`：移动到页面底部（B）。
5. 允许“表格内部横向滚动”兜底（A）。
6. 不能影响 PC 端。

---

## 3.1 设备与宽度验收基线（新增）

为确保“手机+平板都能工作”，本次定义固定验收宽度矩阵（CSS 像素）：

1. 平板横屏：1200、1024
2. 平板竖屏：820、768
3. 手机大屏：430、414
4. 手机主流：390、375、360
5. 手机小屏：320

方向要求：

1. 1200/1024/820/768 需覆盖横竖屏。
2. 430 及以下至少覆盖竖屏。
3. 所有断点必须满足：页面级不横向滚动、表格级可横向滚动、核心操作可触达。

---

## 4. 问题分级与对应范围

### P1（必须修复）

1. 表格在窄屏裁切、列不可见。  
涉及：工作台关注池、实时模拟、历史回放、信号详情。

2. 信号详情多列表格在移动端不可读。  
涉及：`/signal-detail/*` 各指标/投票/阈值表格。

### P2（高优先级）

1. 工具栏控件固定宽+不换行导致溢出。  
涉及：关注池、实时模拟、历史回放、发现/研究/持仓工具栏。

2. 历史回放指标卡强制四列，覆盖响应式。  
涉及：`his-replay-page.tsx` 内联 `gridTemplateColumns`。

### P3（优化级）

1. 次要页面最小宽度控件导致拥挤。  
涉及：发现、研究、持仓页搜索/筛选条。

2. 移动端信息层级弱（表格详情需展开查看）。

---

## 5. 总体设计

## 5.1 响应式分层

1. `>1200px`：原行为保持不变（PC）。
2. `<=1200px`：进入“Compact Table Mode（移动表格模式）”。
3. `<=920px`：沿用现有更强收敛规则，并与本次 `<=1200px` 规则兼容（不冲突）。

## 5.2 移动表格模式（核心）

对业务表格统一使用四段结构：

1. 主行仅保留核心列（示例：代码、名称、关键状态）。
2. 展开行显示次要字段（价格、来源、时间、说明等）。
3. 操作列使用“⋯”弹出菜单（详情、删除、加入候选、加入关注池等）。
4. 外层容器允许 `overflow-x:auto` 作为兜底。

## 5.3 PC 零影响策略

所有新样式和行为都加 `@media (max-width: 1200px)` 或移动模式 class 门控，避免污染 PC：

1. 样式层：仅新增移动端覆盖，不改原 PC 基础值。
2. 组件层：仅在窄屏渲染 `MoreMenu` 与 `ExpandRow` 分支，PC 保持旧渲染。

## 6. 页面级方案

## 6.1 工作台（Workbench）

### 6.1.1 关注池表格

1. 主行保留：勾选、代码、名称、状态（量化状态可合并到状态）。
2. 次要字段（现价、板块、其他说明）进入展开详情。
3. 行内操作（量化/删除等）进入“⋯”菜单。
4. 搜索框和顶部动作栏支持折行，去掉强制不收缩约束。
5. 表格容器允许横向滚动兜底。

### 6.1.2 股票分析区

1. 保持“左侧分析师列表 + 右侧观点”结构，但在 `<=1200px` 自动改单列堆叠。
2. 观点区域保留滚动条（已存在），确保不截断。
3. 不改 PC 排版。

### 6.1.3 Next Step 下沉

在 `<=1200px` 时从右侧栏改为页面底部（紧跟主业务内容后），PC 不动。

## 6.2 实时模拟（live-sim）

1. 信号表格启用“核心列 + 展开详情 + 更多菜单”。
2. 筛选条（股票过滤、动作过滤、分页）改可换行并缩窄控件宽度。
3. 默认 BUY/SELL 筛选逻辑保持不变，仅改善移动端呈现。
4. 字段级合同：
 - 主行核心列：时间、代码、动作、执行结果、更多菜单。
 - 展开详情：策略、置信度、分值、原因、关联任务/来源、原始说明文本。

## 6.3 历史回放（his-replay）

1. 成交明细、信号记录均启用移动表格模式。
2. 任务指标四卡片去掉内联强制四列，交由响应式网格在 `<=1200px` 自动 2/1 列。
3. 筛选条同实时模拟处理。
4. 字段级合同：
 - 成交明细主行核心列：时间、代码、动作、价格、更多菜单。
 - 成交明细展开详情：数量、成交额、信号 ID、任务信息、备注/原因。
 - 信号记录主行核心列：时间、代码、动作、状态、更多菜单。
 - 信号记录展开详情：策略、执行明细、关联指标摘要、详情跳转入口。

## 6.4 信号详情（signal-detail）

1. 所有多列表格（投票、阈值、技术指标、环境指标）统一采用：
 - 主表仅关键列；
 - 次要字段通过展开查看；
 - 容器支持横向滚动兜底。
2. 保持现有业务字段与国际化文案不变。
3. 字段级合同：
 - 投票表主行核心列：维度、投票主体、投票、贡献分。
 - 投票表展开详情：信号分、权重、依据、计算。
 - 决策参数/阈值表主行核心列：参数、值、来源。
 - 决策参数/阈值表展开详情：计算方式、上下文解释。
 - 技术指标表主行核心列：指标、数值、来源。
 - 技术指标表展开详情：说明/计算方式。
 - 环境指标表主行核心列：环境因子、分值。
 - 环境指标表展开详情：说明、来源链路。

## 6.5 发现/研究/持仓（P3）

1. 固定最小宽输入控件在 `<=1200px` 下改为 `min-width:0` 与 `width:100%` 优先。
2. 工具栏允许换行，状态 badge 下沉到下一行，避免挤压主操作按钮。

## 6.6 核心列合同总表（新增）

为避免实现分歧，统一核心列最小集合：

1. 工作台关注池：
 - 主行：勾选、代码、名称、状态、更多菜单
 - 展开：现价、板块、量化状态、扩展说明
2. 实时模拟信号：
 - 主行：时间、代码、动作、执行结果、更多菜单
 - 展开：策略、原因、分值、来源
3. 历史回放成交：
 - 主行：时间、代码、动作、价格、更多菜单
 - 展开：数量、成交额、信号 ID、任务关联
4. 历史回放信号：
 - 主行：时间、代码、动作、状态、更多菜单
 - 展开：策略、执行说明、详情入口
5. 信号详情（各子表）：
 - 主行：关键识别字段 + 数值字段（见 6.4）
 - 展开：计算说明、依据、来源链路

## 7. 组件与样式改造点

## 7.1 样式文件

主改造文件：`ui/src/styles/globals.css`

新增（或重构）移动端规则：

1. `@media (max-width: 1200px)` 下：
 - `.table-shell` 增加 `overflow-x:auto; overflow-y:visible;`
 - `.table` 在移动表格模式允许 `table-layout:auto` 或关键列最小宽。
 - `.toolbar`、`.watchlist-toolbar__cluster`、`.watchlist-toolbar__actions` 解除不必要 `flex-shrink:0` 限制。
 - 输入框与选择器默认 `width:100%`，上限宽度通过 `max-width` 控制。
2. 关注池列宽 class 的移动覆盖（避免固定像素列宽压死）。
3. 移动端 `next-steps` 下沉布局样式。

## 8. 交互细则

## 8.1 行展开

1. 点击“展开”图标或行尾按钮展开。
2. 展开区按“字段名: 值”展示次要信息。
3. 仅当前表内维护展开状态，刷新后可重置（避免状态同步复杂化）。

## 8.2 更多菜单（⋯）

1. 菜单项来源于当前行 actions。
2. 点击菜单项复用原 action handler。
3. 点击外部区域自动关闭。
4. 键盘可访问（Esc 关闭、Tab 可到达）。

## 8.3 触控可用性阈值（新增）

1. 所有可点击控件最小触控区域：`44x44px`。
2. 行展开按钮与“⋯”按钮最小间距：`8px`。
3. 下拉菜单项最小高度：`40px`。
4. 表格分页按钮在 `<=1200px` 下允许换行，不可遮挡筛选控件。

## 8.4 横滚/展开/粘性表头冲突规则（新增）

1. 表头粘性仅在表格内部滚动容器生效，不可吸附到页面级容器。
2. 横向滚动时，展开行保持与当前行同宽，不另起页面级滚动。
3. “⋯”菜单弹层优先向可见区域内展开，避免被 `overflow` 裁剪。
4. 展开行与菜单互斥：
 - 打开菜单时，不自动关闭已展开行；
 - 切换到另一行菜单时，关闭前一行菜单；
 - 行折叠时自动关闭该行菜单。
5. 触屏滑动优先级：
 - 在表格内容区域，单指横滑优先触发表格横向滚动；
 - 点击手势仍可稳定触发行展开与菜单按钮。

## 9. 验收标准（Definition of Done）

### 9.1 全局

1. `>1200px` 页面截图对比不出现结构性差异。
2. `<=1200px` 不出现横向页面级滚动（仅表格内部可横滚）。
3. 所有核心操作可点击且无遮挡。

### 9.2 业务页

1. 工作台关注池：
 - 核心列可读；
 - 次要字段可展开查看；
 - 行操作在“⋯”菜单可用；
 - `Next step` 下沉到底部。

2. 实时模拟/历史回放：
 - 筛选+分页在同屏可操作；
 - 表格不会挤爆；
 - 详情入口不丢失。

3. 信号详情：
 - 多个技术/投票/阈值表可读可滚；
 - 不再出现列内容大面积截断。

4. 发现/研究/持仓：
 - 工具栏不溢出；
 - 搜索/筛选控件在窄屏可完整输入。

### 9.3 手机/平板执行矩阵（新增，必须全部通过）

1. 工作台 `/`：
 - 宽度：1200、1024、820、768、430、390、360、320
 - 检查：关注池表格核心列可读、展开正常、⋯菜单可点、`Next step` 下沉成功
2. 实时模拟 `/live-sim`：
 - 宽度：1200、1024、820、768、430、390、360
 - 检查：筛选+分页可用、信号表不挤爆、详情入口可达
3. 历史回放 `/his-replay`：
 - 宽度：1200、1024、820、768、430、390、360
 - 检查：成交/信号两表可读、四指标卡自动 2/1 列、任务区不溢出
4. 信号详情 `/signal-detail/:id`：
 - 宽度：1200、1024、820、768、430、390、360、320
 - 检查：各子表核心列可读、展开可见、无页面级横滚
5. 发现 `/discover`、研究 `/research`、持仓 `/portfolio`：
 - 宽度：1200、1024、820、768、430、390、360
 - 检查：工具栏不溢出、输入控件完整、按钮可点

通过判定：

1. 任一页面任一宽度出现页面级横滚、操作遮挡、控件不可点击，即判定不通过。
2. 若仅依赖表格内横滚仍不能访问关键操作，判定不通过。
3. 平板竖屏（820/768）与手机主流宽（390/360）必须零阻塞通过。

## 10. 风险与回避

1. 风险：表格改造影响既有行点击逻辑。  
回避：保持原 handler，不改业务 action，只改触发入口。

2. 风险：`<=1200px` 与 `<=920px` 规则冲突。  
回避：先定义 `<=1200px` 通用层，再由 `<=920px` 覆盖更激进策略，避免互相反复重置。

3. 风险：菜单和展开状态引入可访问性退化。  
回避：增加键盘与焦点规则（Esc、Tab、aria 属性）。
