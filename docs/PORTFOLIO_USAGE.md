# 持仓诊断使用指南

本文档只保留当前 `/portfolio` 页面仍然适用的使用说明。

## 1. 当前数据口径

持仓诊断已经不再以独立 `portfolio_stocks.db` 作为股票主表。

当前实现里：

- 股票主对象：`quant_sim.db.stock_universe`
- 登记持仓标签：`registered_position_enabled`
- 个股详情接口：`/api/v1/portfolio_v2/positions/{symbol}`
- 页面快照接口：`/api/v1/portfolio_v2`

## 2. 当前页面能做什么

1. 查看登记持仓列表。
2. 查看组合诊断摘要。
3. 进入个股详情页。
4. 更新持仓成本、数量和备注。
5. 删除登记持仓。
6. 刷新技术指标。
7. 启动或停止持仓诊断调度。

## 3. 当前与实时量化、历史回放的关系

持仓诊断、实时量化、历史回放现在共用统一股票池，但账户和结果口径仍然隔离：

- 持仓诊断看登记持仓字段。
- 实时量化看 live-sim 自己的 `sim_*` 状态。
- 历史回放看 `quant_sim_replay.db` 的 `sim_run_*`。

所以：

- 登记持仓不会自动变成 live-sim 持仓。
- 历史回放不会写回持仓诊断账户。

## 4. 需要注意的旧文档口径

以下旧说法不再适用：

- “持仓数据库是 `portfolio_stocks.db`”
- “直接备份 `portfolio_stocks.db` 就能完整备份当前持仓诊断”

当前如果需要备份与持仓诊断相关的数据，至少要关注：

- `data/quant_sim.db`
- `data/stock_analysis.db`
